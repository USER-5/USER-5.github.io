Why is it like this?

Websites are extremely complicated to build these days. We've gotten to the point where most web developers have layers of abstraction on top of layers of abstraction on top of the actual technology that's used.

And as a web developer myself, I use these layers every day.

These layers are useful for building powerful tools, social media, games, entertainment platforms, and so on - but in my opinion it's unnecessary for a simple personal website.

Then one day I had to create a modern website from scratch, with no access to frameworks or modern tooling. And it was surprisingly capable.

It was then that I realised we've overcomplicated things.

I have always wanted a blog, but never could decide on the right framework. Do I use MkDocs? Wordpress? Something custom? Whatever I choose, I'm locked into it.

Instead, I'm using this site make a point: It doesn't have to be complicated. You don't need to write prose in markdown and compile it into a site. HTML was built for creating web pages - and it's perfectly suitable.

The final piece of inspiration came from this somewhat rude website, which shows just how little CSS is required to make a plain HTML document significantly more reader-friendly.

If you don't like the look of this site, I don't blame you. Feel free to install an extension and add your own styles to your liking. You'll find that there isn't much to override.
